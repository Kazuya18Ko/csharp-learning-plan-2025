# 型と変数のまとめ(Day2)

## `var`の使い方

- `var`は推論型で、**初期化時に型が決まる**
- 変数の宣言に使用され、型を明示的に指定する必要がない
- 明示的に型を指定する必要はないが、**初期値は必須**
- **スコープは宣言されたブロック内のみ**
- 実務では**チームでのコーディング規約**に従って利用する必要がある

```csharp
var x = 10; // int型の変数
var name = "Alice"; // string型の変数
var isActive = true; // bool型の変数
var items = new int[] { 1, 2, 3 }; // int型の配列変数

```

## varと明示的な方指定の使い分け

|状況                                               |推奨される書き方               |理由                                           |
|---------------------------------------------------|-------------------------------|-----------------------------------------------|
|初期値から型が明確にわかる場合                     |`var`                          |冗長的な型記述を避け、可読性を保つため         |
|型が明確でない、又は初見で分かりにくい場合         |明示的な型指定(例:`Person`など)|読みやすさと諸種性を優先するため               |	  
|プリミティブ型(`int`,`string`,`bool`など)を使う場合|`var`でも明示型でもどちらでも可|チームのコーディング規約に準拠する             |
|ラムダ式やLINQクエリなどで型が長くなる場合         |`var`                          |型名が長く可読性が下がるため、`var`で簡潔に書く|

## 注意点
- `var`を使っても、コンパイル時には型が決まる(**動的型付けではない**)
- 初期化せずに`var`を使用するとコンパイルエラーになる
```csharp
var x; // コンパイルエラー: 初期化が必要
```
- `var`は多用しすぎると、コードの可読性が低下する可能性がある
