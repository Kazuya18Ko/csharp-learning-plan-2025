---

## 仕様方針
- タスク（Todo）の追加・表示・完了・削除をサポートするコンソールアプリケーション
- データはメモリ上（`List<TodoItem>`）に保持し、ファイル保存は未対応（発展課題）。
- 処理は以下の構成で責任分離を意識して実装:
  - `Models/`:タスクのデータ構造を定義
  - `Services/`:ビジネスロジック（タスク操作）を管理
  - `Views/`:コンソール出力を管理
  - `Program.cs`:ユーザー入力・処理分岐を担当
